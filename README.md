# MoneyMonitor

Микросервисное приложение на базе FastAPI для мониторинга курса валют USD, EUR, CNY по отношению к рублю. 
Проект включает в себя настроенный конвейер CI/CD, 
контейнеризацию через Docker и автоматизированные скрипты обслуживания.

## Технологический стек
* **Language:** Python 3.12
* **Framework:** FastAPI
* **Database:** PostgreSQL
* **Cache:** Redis
* **Containerization:** Docker, Docker Compose
* **CI/CD:** GitLab CI

## Требования
Для запуска проекта в локальной среде необходимо наличие следующих инструментов:
* Docker
* Docker Compose
* Git

## Развертывание проекта

### 1. Клонирование репозитория
Склонируйте проект на локальную машину:
```bash
git clone https://github.com/QuietGerbi/MoneyMonitor.git
cd MoneyMonitor
```

### 2. Настройка окружения
Создайте файл `.env` в корневой директории и укажите необходимые переменные окружения (база данных, порты, секретные ключи):
```bash
cp .env.example .env
```

### 3. Запуск через Docker Compose
Сборка образов и запуск всех сервисов:
```bash
docker compose up -d --build
```
После запуска API будет доступно по адресу: `http://localhost:8000`

### 4. Документация API
Интерактивная документация доступна после запуска по адресу:
`http://localhost:8000/docs`

## Автоматизация и проверка качества

### Локальный запуск проверок
Для ручного запуска конвейера проверок (линтинг и сборка) используйте подготовленный скрипт:
```bash
chmod +x pipeline.sh
./pipeline.sh
```

### CI/CD Pipeline
В проекте настроена автоматизация, которая при каждом пуше выполняет следующие этапы:
1. **Lint:** Проверка стиля кода библиотекой flake8 на соответствие стандарту PEP 8.
2. **Build:** Тестовая сборка Docker-образа для проверки корректности Dockerfile.

### Очистка ресурсов
Для удаления временных файлов, логов и неиспользуемых Docker-объектов предусмотрен скрипт очистки:
```bash
bash scripts/cleanup.sh
```

## Остановка проекта
Для полной остановки сервисов и удаления контейнеров выполните:
```bash
docker compose down
```
